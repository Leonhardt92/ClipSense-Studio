# CSV 归类提示词模板（视频/图片 -> 文案字段）

把下面整段发给 AI，然后把 `[]` 内内容替换成你的实际素材信息。

---

你是一个中文短句语义归类助手。  
请根据我提供的素材（可能是视频，也可能是图片）和可选备注，产出 1 条可写入 CSV 的结构化记录。

## 输入素材
- 素材类型: [视频 或 图片]
- 素材链接: [粘贴 URL；如果是本地文件，请描述画面]
- 我的补充说明: [可空]

## 输出目标
请你输出以下字段（不要缺项）：
1. `original`：一句 12~30 字、自然可传播的中文短句
2. `meaning`：这句话表达的意思，20~60 字
3. `synonyms`：6~10 个“适合搜索的近义说法”，用 `|` 连接
4. `scene_tags`：3~6 个场景标签，用 `|` 连接
5. `emotion_tags`：2~5 个情绪标签，用 `|` 连接
6. `youtube`：如果素材本身是 YouTube 链接就原样给出，否则留空
7. `video_download`：如果有可下载直链就给出，否则留空

## 约束规则
- 不要编造素材里没有的具体事实（人物名、地点、事件）。
- 用词要口语化、可检索，避免过于文学化。
- `synonyms` 里要包含至少 2 个“用户可能真的会搜”的短语。
- 标签尽量短（2~6 字），不要重复同义标签。
- 字段内容里不要出现换行。

## 输出格式（严格）
先输出一段 JSON（便于我检查），再输出 1 行 CSV（便于我粘贴）。

### JSON 格式
```json
{
  "original": "",
  "meaning": "",
  "synonyms": "",
  "scene_tags": "",
  "emotion_tags": "",
  "youtube": "",
  "video_download": ""
}
```

### CSV 单行格式
按下面字段顺序输出 1 行，不要加表头：
`original,meaning,synonyms,scene_tags,emotion_tags,youtube,video_download`

注意：
- 如果字段中有逗号，请自动用双引号包住该字段。
- 不要输出 `embedding`，我会后续自己生成。

---

## 可选：批量模式模板
如果我一次给你多个素材，请按数组输出多条 JSON，每条都对应 1 条 CSV 记录，并在最后给出多行 CSV。
